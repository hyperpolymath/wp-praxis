## Becoming a Maintainer

WP Praxis uses the **Tri-Perimeter Contribution Framework (TPCF)** for graduated access.

### Path to Component Maintainer (Perimeter 2)

**Criteria**:
1. **Expertise**: Demonstrated deep knowledge of the component
2. **Contributions**: 10+ merged PRs in the component area
3. **Consistency**: 6+ months of regular contribution
4. **Review Participation**: Actively reviews others' PRs
5. **Community**: Helps others in discussions and issues
6. **Alignment**: Shows alignment with project values

**Responsibilities**:
- Review and approve PRs for your component
- Maintain component documentation
- Triage issues for your component
- Mentor new contributors
- Coordinate with other component maintainers
- Participate in architectural discussions

**Benefits**:
- Direct commit access to development branches
- Listed on MAINTAINERS.md
- Vote on component-specific decisions
- Invitation to maintainer meetings (when established)

### Path to Core Maintainer (Perimeter 1)

**Criteria** (tentative, not yet established):
1. **Component Maintainer** for 12+ months
2. **Cross-Component Knowledge**: Understanding of multiple components
3. **Leadership**: Mentored other contributors
4. **Governance**: Participated in major decisions
5. **Trust**: Demonstrated judgment and responsibility
6. **Availability**: Committed to long-term project health

**Additional Responsibilities**:
- Release management
- Security incident response
- Roadmap planning
- Conflict resolution
- Budget decisions (if applicable)

### Code Review Standards

When reviewing PRs, check for:

1. **Functionality**: Does it work as intended?
2. **Tests**: Are there tests? Do they pass?
3. **Documentation**: Are docs updated?
4. **Code Quality**: Follows language standards?
5. **Security**: No security vulnerabilities?
6. **Breaking Changes**: Are they justified and documented?
7. **Performance**: No significant regressions?
8. **Accessibility**: UIs are accessible?

### Communication

Maintainers are expected to:

- **Be Responsive**: Reply to mentions within 48 hours
- **Be Transparent**: Explain decisions publicly
- **Be Respectful**: Follow Code of Conduct
- **Be Patient**: Help beginners learn
- **Be Collaborative**: Seek consensus on major decisions

## Stepping Down

Maintainers can step down at any time:

1. **Notify**: Email other maintainers or open an issue
2. **Transition**: Help find/train a replacement (if possible)
3. **Remove Access**: Maintainer permissions will be revoked
4. **Alumni Status**: Listed as "Emeritus Maintainer" if desired

## Inactive Maintainers

If a maintainer becomes inactive (no activity for 6+ months):

1. **Reach Out**: Email to check status
2. **Grace Period**: 30 days to respond
3. **No Response**: Move to emeritus status
4. **Return**: Welcome to rejoin anytime
